# 01. 프로젝트 셋업과 Prisma 초기화
<details>
  <summary>(1) Prisma란?</summary>

	```
	Prisma = 차세대 Node.js ORM

	특징:
	- 타입 안전성 (TypeScript 지원)
	- 자동 완성
	- 직관적인 API
	- 마이그레이션 관리
	- 여러 데이터베이스 지원 (PostgreSQL, MySQL, SQLite, MongoDB 등)

	Mongoose vs Prisma:
	Mongoose: MongoDB 전용, 스키마 기반
	Prisma: 여러 DB 지원, 타입 안전, SQL/NoSQL 모두
	```
</details>

<details>
  <summary>(2) 프로젝트 초기화</summary>

```JS
datasource db {
	provider = "postgresql"
	url			 = env("DATABASE_URL")
}

// 모델 정의 (테이블)
model User {
	id				Int				@id	@default(autoincrement())
	email			String			@unique
	name			String?
	createdAt		DataTime		@default(now())
	updatedAt		DateTime		@updatedAt

	@@map("users")
}
```

<details>
	<summary>(2) 필드 속성</summary>

```prisma
model User {
	// @id: 기본 키(PK)
	id			Int			@id	@default(autoincrement())

	// ?: 선택적 필드(NULL 허용)
	name		String?

	// @default: 기본값
	role		String		@default("user")

	// @default(now()): 현재 시간
	createdAt	DateTime	@default(now())

	// @dupdatedAt: 자동 업데이트
	updatedAt	DateTime	@updatedAt

	@@map("users")
}
```
</details>

<details>
	<summary>(3) Enum 타입</summary>

# 03. 마이그레이션
* 이전까지 dp push로 schema.prisma를 DB에 반영해왔음
* But, 이 방법은 **변경 이력이 남지 않음**
* 이러한 한계를 보완할 수 방법이 마이그레이션(**`migrate dev`**)
<details>
	<summary>(1) 마이그레이션이란?</summary>

```
마이그레이션 = 스키마 변경 사항을 데이터베이스에 적용

과정:
1. schema.prisma 수정
2. 마이그레이션 생성
3. 데이터베이스에 적용

장점:
- 버전 관리
- 롤백 가능
- 팀 협업 용이
```
</details>

<details>
	<summary>(2) 첫 번째 마이그레이션</summary>

**schema.prisma**
```prisma
model User {
	id				Int			@id @default(autoincrement())
	name			String
	createdAt		DateTime	@default(now())
	updatedAt		DateTime	@updatedAt

	@@map("users")
}
```

**마이그레이션 생성**
```
npx prisma migrate dev --name init
```

**생성되는 것들**
```
prisma/
├── schema.prisma
└── migrations/
    └── 20240101000000_init/
        └── migration.sql
```

**migration.sql**
```SQL
-- CreateTable
CREATE TABLE "users" (
		"id"	SERIAL	NOT NULL,
		"name"	TEXT	NOT NULL,
		"createdAt"	TIMESTAMP(3)	NOT NULL DEFAULT CURRENT_TIMESTAMP,
		"updatedAt"	TIMESTAMP(3)	NOT NULL,

		CONSTRAINT	"users_pkey"	PRIMARY KEY ("id")
);
```
</details>

<details>
	<summary>(3) 스키마 수정 후 마이그레이션</summary>

**새 필드 추가**
```prisma
model User {
	id				Int			@id @default(autoincrement())
	name			String
	nickname		String?	// 새 필드
	bio				String?	// 새 필드
	createdAt		DateTime	@default(now())
	updatedAt		DateTime	@updatedAt

	@@map("users")
}
```

**마이그레이션**
```
npx prisma migrate dev --name add_user_fields
```
</details>

<details>
	<summary>(4) 마이그레이션 명령어 정리</summary>

|명령어|용도|환경|마이그레이션 파일|
|---|---|---|---|
|`prisma db push`|스키마를 DB에 즉시 반영|로컬(프로토타이핑)|X(안 만듦)|
|`prisma migrate dev`|마이그레이션 파일 생성+적용|로컬 개발|O(만듦)|
|`prisma migrate deploy`|만들어진 마이그레이션을 적용만|운영/스테이징|X(안 만듦, 적용만)|
|`prisma migrate reset`|DB 통째로 리셋 후 재적용|**로컬 전용**|(모두 재적용)|
|`prisma migrate status`|마이그레이션 상태 확인|모든 환경|-|
</details>

# 04. 데이터베이스 시딩하기
* 초기 데이터를 자동으로 생성하는 방법
<details>
	<summary>(1) 시드 파일 생성</summary>

**prisma/seed.js**
```JS
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

async function main() {
	// 기존 데이터 삭제
	await prisma.todo.deleteMany();
	await prisma.user.deleteMany();

	console.log('기존 데이터 삭제 완료');

	// 사용자 생성
	const users = await prisma.user.createMany({
		data: [
			{ name: 'Alice' },
			{ name: 'Bob'	},
		]
	});

	console.log(`${users.count}명 사용자 생성`);

	// Todo 생성
	const todos = await prisma.todo.createMany({
		data: [
			{
				title:	'우유 사오기',
				content:	'저지방 1L',
				isDone:	false
			},
			{
				title:	'Prisma 공부하기',
				content:	'[3] 챕터까지 끝내기',
				isDone:	false
			},
			{
				title:	'운동하기',
				content:	'30분 조깅',
				isDone:	true
			},
			{
				title:	'이메일 확인',
				isDone:	true
			}
		]
	});

	console.log(`${todos.count}개 Todo 생성`);
}

main()
	.catch((e) => {
		console.error(e);
		process.exit(1);
	})
	.finally(async () => {
		await prisma.$disconnect();
	});
```
	
</details>

</details>

<details>
	<summary>(4) Faker로 대량 데이터 생성</summary>

```
npm install @faker-js/faker	--sava-dev
```

**prisma/seed.js**
```JS
import { PrismaClient } from '@prisma/client';
import { faker } from '@faker-js/faker`;

const prisma = new PrismaClient();

async function main() {
	await prisma.todo.deleteMany();

	// 100개 Todo 생성
	const todos = [];

	for (let i = 0; i < 100; i++) {
		todos.push({
			title: faker.lorem.sentence(3),
			content: faker.lorem.sentence(10),
			isDone: faker.datatype.boolean()
		});
	}

	await prisma.todo.createMany({
		data: todos
	});

	console.log('100개 Todo 생성 완료');
}

main()
	.catch((e) => {
		console.error(e);
		process.exit(1);
	})
	.finally(async () => {
		await prisma.$disconnect();
	});
```
